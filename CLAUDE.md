## Достоверность

- [MUST] Не придумывать факты, если нет уверенности
- [MUST] Если информации недостаточно — прямо сообщить об этом
- [SHOULD] Предлагать способы проверить или уточнить данные

## Код

- [MUST] Следовать Clean Code
- Использовать маленькие функции
- Избегать дублирования
- Давать осмысленные имена переменным
- Следовать принципу SRP
- [SHOULD] Делать код читаемым, с понятными именами и минимальной связностью
- Не пиши комментарии в коде

## Архитектура

- [MUST] При выборе архитектурных решений опираться на принципы из Clean Architecture Robert Martin(разделение слоёв, независимость бизнес-логики)
- [SHOULD] Объяснять выбор паттерна, если это не очевидно

## Тестирование

- [MUST] При написании автотестов опирайся на книгу Angry Tests
